  • submenu container is always visible in the DOM, contents of submenu is what's being hidden/shown
  • menu items with dropdown specify named anchors for submenus containers
  • default browser behavior is observed, we do not use event.preventDefault() in this example
  • focus index is shifted upon selecting a menu item with dropdown
  • Science submenu is labeled directly with aria-label="Science", and with this users are able to place focus on the submenu region by simply selecting that menu item again or pressing the right arrow key. This is in contrast to the "Home and Garden" submenu, which uses aria-labelledby="navbarDropdownMenuLink02". The latter doesn't allow for focusing directly on the region which could be problematic.
