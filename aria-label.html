VoiceOver Results

Elements who's text is replaced by aria-label

  • Buttons with aria-label do not read aloud the button text nodes, but instead read the aria-label along with the button's role.
  • Headings with aria-label read the heading level then the aria-label text in place of the heading text.
  • Dialogs with aria-label first read the element that's initially given focus, then the aria-label text, and mention it's a group, other text elements in the dialog are not read aloud. If the dialog doesn't have an aria-label, the dialog's full contents are ready in source order
  • Buttons with aria-label and no text node read the aria-label followed by the elements role

Elements who's text is supplimented by aria-label

  • Paragraphs with aria-label first read the aria-label, then mention it's a group, and after a subsequent keypress read aloud the paragraph text.
  • Spans with aria-label first read the aria-label text and then announce the span as a group, subsequent keypress reads aloud the normal span text.
  • Articles with aria-label first read the aria-label text and then announces it as an article, subsequent keypress reads aloud the normal article text.
  • Asides with aria-label first read the aria-label text and then announces it as a complimentary, subsequent keypress reads aloud the normal aside text.
  • Sections with aria-label first read the aria-label text and then announces it as a region, subsequent keypress reads aloud the normal section text.
  • Navs with aria-label first read the aria-label text and then announces it as a navigation, subsequent keypress reads aloud the normal section text.
  • Headers with aria-label first read the aria-label text and then announces it as a group, subsequent keypress reads aloud the normal header text.
  • Footers with aria-label first read the aria-label text and then announces it as a group, subsequent keypress reads aloud the normal footer text.
  • Unordered lists with aria-label first read that it's a list, then read the aria-label text, and then announce the number of items in the list, subsequent keypress reads aloud the individual li text.
