Minimal notification container with aria-atomic, and role attributes

  • Alert 1
  • Alert 2
  • Alert 3
  • each notification is injected into and removed from a single notifications container which is designated as an aria-live region
  • the notifications container holds the role="alert" attribute
  • notifications container holds the aria-atomic="false" attribute, to ensure only new changes to the region are announced
  • notification container is a basic division element
  • each notification has a slightly different sr-only text, which seems to help ensure the entire notification (including warning/alert prefix) is read each time the notification is launched, but it is not flawless
  • notifications are not dismissed automatically, and require user action to be removed
